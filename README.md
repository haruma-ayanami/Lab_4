Лабораторно-практическая работа №4

Порождающие паттерны проектирования: Синглтон, Прототип и Строитель

Выполнил: Ткачев А. Д.

Цель работы:
Изучить порождающие паттерны проектирования «Банды четырёх (GoF)» на примере разработки приложения-конфигуратора компьютерных систем. Освоить на практике паттерны Строитель (Builder) для пошагового конструирования сложных объектов, Прототип (Prototype) для клонирования объектов с различной глубиной копирования и Одиночка (Singleton) для обеспечения единственного экземпляра класса.

Задание:

Разработать консольное приложение, моделирующее конфигуратор компьютерных систем.

Программа должна позволять:
1. Создавать объекты класса Computer, содержащие следующие компоненты:
- процессор (строка);
- объём оперативной памяти (число);
- видеокарта (строка);
- список дополнительных комплектующих (список строк).
2. Использовать паттерны Строитель и Фабричный метод для создания различных конфигураций компьютеров:
- офисный компьютер;
- игровой компьютер;
- домашний компьютер.
3. Использовать паттерн Prototype для клонирования существующих конфигураций и продемонстрировать различие между поверхностным и глубоким копированием.
4. Использовать паттерн Singleton для создания реестра прототипов PrototypeRegistry, который хранит предопределённые конфигурации компьютеров и предоставляет доступ к ним.

Теоретическое обоснование

Порождающие паттерны проектирования предназначены для решения задач, связанных с созданием объектов. Они позволяют отделить процесс создания объектов от их использования, что делает программную архитектуру более гибкой и расширяемой. Паттерн Строитель (Builder) используется для пошагового создания сложных объектов. Он позволяет создавать объект постепенно, задавая его характеристики с помощью отдельных методов. Такой подход делает код более читаемым и удобным для модификации.
Паттерн Прототип (Prototype) позволяет создавать новые объекты путём копирования уже существующих экземпляров. Это удобно, когда создание объекта «с нуля» является сложным или требует значительного количества параметров. Важным аспектом данного паттерна является различие между поверхностным и глубоким копированием объектов. Поверхностное копирование копирует значения полей объекта, однако ссылочные поля продолжают ссылаться на те же объекты. Глубокое копирование создаёт новые экземпляры всех вложенных объектов, благодаря чему изменения в копии не влияют на исходный объект.
Паттерн Одиночка (Singleton) гарантирует существование только одного экземпляра класса в системе. Такой подход используется для объектов, которые должны быть глобально доступны и иметь единственное состояние в приложении. Для обеспечения потокобезопасности используется механизм ленивой инициализации объекта.
Использование данных паттернов позволяет повысить гибкость архитектуры программы, уменьшить связанность компонентов системы и упростить дальнейшее расширение функциональности.

Описание выполненных действий

Для выполнения лабораторной работы было разработано консольное приложение, моделирующее конфигуратор компьютерных систем. Сначала был создан класс Computer, содержащий основные характеристики компьютера: процессор, объём оперативной памяти, видеокарту и список дополнительных комплектующих. Также был реализован метод для вывода информации о конфигурации компьютера. Далее был реализован паттерн Строитель. Для этого был создан класс ComputerBuilder, который позволяет последовательно задавать параметры конфигурации компьютера. Методы строителя позволяют устанавливать процессор, объём оперативной памяти, видеокарту и дополнительные компоненты. После задания всех параметров создаётся готовый объект компьютера. Затем был реализован Фабричный метод для создания стандартных конфигураций компьютеров. Для этого был создан интерфейс фабрики, определяющий метод создания компьютера. На основе этого интерфейса были реализованы конкретные фабрики, создающие офисный, игровой и домашний компьютер. После этого был реализован паттерн Prototype. В классе Computer были добавлены методы поверхностного и глубокого копирования. Это позволило продемонстрировать различие между этими подходами и показать влияние изменений в копии на исходный объект. На последнем этапе был реализован паттерн Singleton. Для этого был создан класс реестра прототипов, который хранит заранее подготовленные конфигурации компьютеров. Реестр реализован таким образом, чтобы существовал только один экземпляр данного класса. Внутри реестра хранится набор прототипов, которые можно получать по ключу. Аналогичные действия по созданию и настройке конфигураций выполнены для всех типов компьютеров.

Результат выполненной работы

В результате выполнения лабораторной работы было создано консольное приложение-конфигуратор компьютерных систем. Проверка работы программы показала, что: Создание компьютера с использованием паттерна Builder позволяет удобно задавать параметры конфигурации. Использование Factory Method позволяет быстро создавать типовые конфигурации компьютеров без изменения клиентского кода. Реализация Prototype демонстрирует различие между поверхностным и глубоким копированием объектов. При поверхностном копировании изменения в списке дополнительных компонентов могут повлиять на исходный объект, тогда как при глубоком копировании создаётся полностью независимая копия. Паттерн Singleton обеспечивает существование единственного экземпляра реестра прототипов. Проверка показала, что все обращения к реестру возвращают один и тот же объект. Полученные результаты соответствуют ожидаемому поведению программы и подтверждают корректность реализации паттернов проектирования.
